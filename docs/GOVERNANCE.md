# Governance

## 方針

このプロジェクトは、公共データ活用を目的とした OSS として運用します。  
変更は「透明性」「再現性」「継続性」を満たすことを重視します。

## 役割

- Maintainer
  - マージ権限、リリース判断、運用ルール更新
- Reviewer
  - コード・データ・ドキュメントのレビュー
- Contributor
  - Issue 作成、実装、ドキュメント改善

## 意思決定

- 小規模変更
  - PR レビュー 1 件以上でマージ可
- 仕様変更・破壊的変更
  - Issue で背景と代替案を整理後に実装
- データスキーマ影響変更
  - `docs/DATA_PIPELINE.md` の更新を必須とする

## リリースと互換性

- `main` は常にデプロイ可能状態を維持
- ページ URL / パス構造の変更は互換性影響を明記
- `base` パス変更は破壊的変更として扱う

## 記録の残し方

- 実装意図: PR 本文
- 運用ルール: `docs/` 配下
- データ変化: evidence JSON と更新 PR

## コミュニケーション

- バグや提案は Issue ベースで管理
- セキュリティは `SECURITY.md` 手順に従う
- 行動規範は `CODE_OF_CONDUCT.md` を適用
