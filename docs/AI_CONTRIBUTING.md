# AI Contributing Guide

## 目的

AI 支援によるコントリビューションを、再現可能でレビュー可能な形に保つための補助ガイドです。

## 提出時に必要な情報

- 使用した AI ツール名
- AI が主に生成したファイル
- 人間が確認・修正したポイント
- 未解決の懸念点

## 推奨フロー

1. まず小さな単位で変更する
2. 生成コードを人間が読んで責任を持って確定する
3. `npm run build` で最低限の動作確認を行う
4. PR 本文に「AI 生成部分」と「人手確認範囲」を明示する

## 禁止事項

- 外部データの出典を隠すこと
- 生成物を根拠なく手動改ざんすること
- 未検証のまま自動マージを前提にすること

## データプロジェクト特有の注意

- 取得元 URL やライセンス情報を改変する場合は根拠を示す
- evidence JSON の `sha256` や `timestamp` は取得結果に基づく
- CSV ヘッダー変更を無視して固定列前提コードを増やさない
